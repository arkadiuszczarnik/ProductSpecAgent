# Koog 0.8.0 Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the JetBrains Koog dependency from 0.7.3 to 0.8.0 and wire the newly added GPT-5.4 Mini/Nano models into the SMALL/MEDIUM tiers.

**Architecture:** Two separable commits on branch `chore/koog-0.8-upgrade`. Commit 1 is a pure version bump in Gradle. Commit 2 adds two new aliases to `OpenAiModelResolver`, switches the production tier mapping, and extends the resolver's unit test. Each commit is independently revertible.

**Tech Stack:** Kotlin 2.3, Gradle, Spring Boot 4, JetBrains Koog (`ai.koog:koog-spring-boot-starter`), JUnit 5 + AssertJ.

**Spec:** `docs/superpowers/specs/2026-05-03-koog-0.8-upgrade-design.md`

**Branch:** `chore/koog-0.8-upgrade` (already created with the spec commit)

- [ ] **Step 3: Add the two new branches to the resolver**

```kotlin
package com.agentwork.productspecagent.agent

import ai.koog.prompt.executor.clients.openai.OpenAIModels
import ai.koog.prompt.llm.LLModel

fun resolveOpenAiModel(name: String): LLModel = when (name) {
    "gpt-4o" -> OpenAIModels.Chat.GPT4o
    "gpt-4o-mini" -> OpenAIModels.Chat.GPT4oMini
    "gpt-4.1" -> OpenAIModels.Chat.GPT4_1
    "gpt-4.1-mini" -> OpenAIModels.Chat.GPT4_1Mini
    "gpt-4.1-nano" -> OpenAIModels.Chat.GPT4_1Nano
    "gpt-5-nano" -> OpenAIModels.Chat.GPT5Nano
    "gpt-5-mini" -> OpenAIModels.Chat.GPT5Mini
    "gpt-5" -> OpenAIModels.Chat.GPT5
    "gpt-5-2" -> OpenAIModels.Chat.GPT5_2
    "gpt-5-2-pro" -> OpenAIModels.Chat.GPT5_2Pro
    "gpt-5.4-mini" -> OpenAIModels.Chat.GPT5_4Mini
    "gpt-5.4-nano" -> OpenAIModels.Chat.GPT5_4Nano
    else -> throw IllegalStateException("Unknown OpenAI model id: $name")
}
```

If Step 2 turned up different symbol names, replace `GPT5_4Mini` / `GPT5_4Nano` here with whatever the jar actually exposes.
