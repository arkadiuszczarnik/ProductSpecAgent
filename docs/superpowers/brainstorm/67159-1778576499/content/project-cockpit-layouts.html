Projekt-Cockpit nach dem Wizard

Welche Informationsarchitektur fuehlt sich richtig an, ohne zum ueberladenen Dashboard zu werden?

AStatus zuerst

Delivery Cockpit

Oben Fortschritt und Tests, darunter Done Features, offene Punkte und Feature-Erfassung.

BFeature zuerst

Feature Workbench

Links Feature-Liste, rechts der aktive Feature-Kontext mit Tests, offenen Punkten und Design-Aktionen.

CNaechste Aktion zuerst

Decision Cockpit

Eine priorisierte Arbeitsliste fuehrt durch Done-Review, neue Features und Design-Erzeugung.

Meine Empfehlung: B, weil Product Owner meist an einzelnen Features arbeiten, aber Tests und offene Punkte direkt im Kontext sehen muessen.
