A: Graph oben, Formular unten (Split)

Rete.js Graph kompakt oben als Navigation. Darunter das gefuehrte Formular fuer den aktiven Schritt. Klick auf Node wechselt das Formular.

Pros

  • Graph und Form immer sichtbar
  • Klarer visueller Zusammenhang

Cons

  • Graph-Bereich wird klein
  • Wenig Platz fuer lange Formulare

B: Wizard mit Step-Indicator (empfohlen)

Rete.js Graph wird durch einen horizontalen Step-Indicator ersetzt. Der gesamte Hauptbereich wird zum Formular — viel Platz fuer strukturierte Eingaben. Jeder Schritt hat eigene Felder.

Pros

  • Maximaler Platz fuer Formulare
  • Klarer Wizard-Flow
  • Schritt-fuer-Schritt gefuehrt

Cons

  • Rete.js Graph nicht mehr sichtbar
  • Weniger "spielerisch"

C: Umschaltbar (Graph / Formular / Preview)

Tab-Leiste im Hauptbereich: Graph-View, Formular-View, Spec-Preview. Formular hat eine vertikale Step-Liste links (inkl. Features, Architektur, Backend, Frontend) und das aktive Formular rechts.

Pros

  • Rete.js Graph bleibt erhalten
  • Vertikale Step-Liste zeigt ALLE Schritte (inkl. Features, Arch, Backend, Frontend)
  • Spec-Preview als dritter Tab

Cons

  • Komplexer zu implementieren
  • Mehr Klicks zum Wechseln
