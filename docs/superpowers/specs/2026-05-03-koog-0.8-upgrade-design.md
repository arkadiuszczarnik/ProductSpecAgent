# Koog 0.8.0 Upgrade — Design

**Datum:** 2026-05-03
**Branch:** `chore/koog-0.8-upgrade`
**Scope:** Library-Bump `0.7.3 → 0.8.0` plus Aufnahme der neuen GPT-5.4 Mini/Nano Modelle.

## Motivation

JetBrains Koog 0.8.0 wurde am 2026-04-10 veröffentlicht. Für unseren Stack relevant:

- **Bugfix #1779** — IDs für Reasoning-Messages, behebt 400er von OpenAI bei reasoning-fähigen Modellen (GPT-5*).
- **Bugfix #1835** — fehlende Prompt-Messages bei aktiviertem Chat-Memory-Feature.
- **Bugfix #1825** — korrekte `maxOutputTokens` für Claude Opus 4.6 (für uns nicht direkt relevant, aber Konsistenz).
- **Neue Modelle (#1837)** — `GPT-5.4Mini` und `GPT-5.4Nano` werden als neue Modell-Optionen über `OpenAIModels.Chat.*` exponiert.

Die neuen Modelle sollen unsere Tiers SMALL und MEDIUM ersetzen; LARGE bleibt auf `gpt-5-2`, da kein 5.4-Pro-Pendant existiert.

## Scope

### In Scope

1. **Version Bump** der Koog-Dependency von `0.7.3` auf `0.8.0`.
2. **Resolver-Erweiterung** um zwei neue Modell-Aliasse (`gpt-5.4-mini`, `gpt-5.4-nano`).
3. **Tier-Mapping** in der Production-Config: SMALL und MEDIUM auf die neuen 5.4-Modelle umstellen.

### Explizit Out of Scope

- Adoption neuer 0.8.0-Features (Spring AI Integration, DataDog Observability, ChatHistoryProvider, Native Structured Output für Claude, Mermaid-Diagramme).
- Verhaltensänderungen an `KoogAgentRunner`, `IdeaToSpecAgent`, `DecisionAgent`, `PlanGeneratorAgent`, `FeatureProposalAgent`.
- Anpassungen an `AgentModelRegistry`-Logik.
- Test-Config (`backend/src/test/resources/application.yml`) bleibt auf `gpt-4o-mini`, da Tests gegen Mock-`PromptExecutor` laufen, nicht gegen OpenAI.

Der erwartete Kotlin-Symbolname ist `OpenAIModels.Chat.GPT5_4Mini` / `GPT5_4Nano` (Konvention der bestehenden Konstanten wie `GPT5_2Pro`). Falls die tatsächlichen Symbole abweichen, wird der erste Compile in Commit 2 das aufdecken; gegebenenfalls über context7 vorab verifizieren.

## Nicht-betroffene Aufrufflächen

Folgende Koog-API-Punkte werden im Code verwendet und sind laut Release Notes unverändert:

- `AIAgent(promptExecutor, systemPrompt, llmModel)` (`KoogAgentRunner.kt:23`)
- `PromptExecutor` als abstrakte Klasse mit `execute`, `executeStreaming`, `moderate`, `close` (`KoogAgentRunnerTest.kt:62`)
- `OpenAIModels.Chat.GPT*` Konstanten (`OpenAiModelResolver.kt`)
- `LLModel`, `Prompt`, `Message`, `ResponseMetaInfo`, `StreamFrame`, `ModerationResult`, `ToolDescriptor`

Der einzige genannte Breaking Change in 0.8.0 betrifft `LLMProvider`-Singletons (#1800). Wir verwenden `LLMProvider` nicht direkt — nur `LLModel` und die `OpenAIModels.Chat.*`-Konstanten. Kein Impact erwartet.

## Risiken & Mitigation

| Risiko | Mitigation |
| --- | --- |
| GPT-5.4-Konstantennamen weichen ab | Erster Compile in Commit 2 zeigt das; ggf. context7-Lookup vor dem Schreiben |
| Spring-Boot-Starter Property-Pfad `ai.koog.openai.api-key` umbenannt | Smoke-Test fängt das auf (Bean-Binding-Fehler beim Start) |
| `PromptExecutor`-API in 0.8.0 leicht geändert (LLMClient-Decoupling, #1742) | `./gradlew test` deckt das durch den `CapturingExecutor` in `KoogAgentRunnerTest` ab |
| Reasoning-Message-Bugfix ändert Request-Format | Smoke-Test deckt das auf |

## Verifikationsstrategie

### Nach Commit 1 (Bump)

1. `./gradlew build` muss grün sein. Compile + alle bestehenden Tests laufen durch.
2. Kein Smoke-Test in diesem Schritt — Modellauswahl unverändert.

### Nach Commit 2 (Wiring)

1. `./gradlew test` grün, inkl. der zwei neuen Resolver-Tests.
2. **Lokaler Smoke-Test:**
   - `./start.sh` startet Backend + Frontend.
   - Im Frontend ein neues Projekt anlegen, IDEA-Wizard-Step ausfüllen und absenden.
   - Mindestens einen MEDIUM-Agenten anstoßen, damit das neue `gpt-5.4-mini`-Modell tatsächlich getroffen wird (z. B. einen Wizard-Step bis FEATURES durchklicken — `feature-proposal` ist MEDIUM, oder `decision`-Markup auslösen).
   - Erwartet: keine 5xx im Backend-Log, Antwort kommt zurück.

### Rollback

Jeder Commit ist einzeln revertierbar. Falls nur die Modellumstellung Probleme macht, wird Commit 2 revertiert und der reine Bump (Commit 1) bleibt.

## Erfolgs-Kriterien

- `backend/build.gradle.kts` referenziert `ai.koog:koog-spring-boot-starter:0.8.0`.
- `OpenAiModelResolver` löst `gpt-5.4-mini` und `gpt-5.4-nano` auf die entsprechenden Koog-Konstanten auf.
- Production-Config bindet SMALL/MEDIUM auf die 5.4-Modelle.
- `./gradlew build` und `./gradlew test` grün.
- Lokaler Smoke-Test trifft mindestens einen GPT-5.4-Aufruf erfolgreich.
- Branch `chore/koog-0.8-upgrade` enthält genau zwei Commits, jeder einzeln revertierbar.
